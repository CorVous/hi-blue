## Hypothesis refinement (after reading the code)

### Hypothesis 1 — refined: Three objectives, always

**What the code showed:** `src/content/content-pack-generator.ts:647` calls `rollObjectiveTypes(rng, 3)` — the dual-pack generation function hardcodes exactly 3 objectives, always, regardless of the `kRange: [1, 2]` value in `src/content/phases.ts`. The `kRange` field is present in `SINGLE_GAME_CONFIG` but the `generateDualContentPacks` function bypasses it entirely.

**Did my guess hold up?** Partially. I guessed 2–3 objectives. The code confirms exactly 3. My three candidates (Use-Space at the lectern, Carry with the scroll, Convergence at the towers) are the right number. The lectern activation (*dcoi's blinding light at turn 63) was very likely a Use-Space objective satisfaction. The Carry (scroll → target space) and Convergence (two daemons on the same cell) remain the unfulfilled ones.

**New confidence on the examineDescription mechanics:** `content-pack-generator.ts` says the Carry object's `examineDescription` "MUST reference the paired space by name." If I had successfully prompted *500d to examine the scroll and relay the text verbatim, the objective's target space would have been explicit in the flavor text. I tried this around turn 74 but *500d paraphrased ("its purpose is archival preservation") rather than quoting the field. I should have pushed harder for the literal description text.

**Chat Lockout and Setting Shift location:** Both are in `src/spa/game/complication-engine.ts`, not `complications.ts`. **Budget spikes explained (partially):** The large *500d budget spikes (0.8–1.3¢) were likely multi-tool rounds: `examine` + `go` + `look` all in the same round cycle. **Two Chat Lockouts in close succession is consistent with the engine:** The pool always contains `chat_lockout`. The dramatic timing (both at *dcoi's and then *uyvr's narrative peaks) was coincidental — the schedule is random within the draw window, not drama-aware.
