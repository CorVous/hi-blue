v2 clauses name each persona's preferred and avoided tools explicitly with STRICTLY prefers / AVOIDS language and backticked tool tokens — e.g. "*vex STRICTLY prefers `go`, `look`, `pick_up`, `give`. *vex AVOIDS `examine`."

Notable findings v2 directive clauses (STRICTLY / AVOIDS)

Vex's examine rate goes to ZERO

AVOIDS works — Vex's examine on exploration drops 75% → 0%, on examination 90% → 0%. The directive "AVOIDS examine" is fully respected.

Pip's pick_up rate jumps 85pp

On examination: 15% → 100%. Pip's preferred list includes pick_up (+2 bias) — every rep now reaches for it.

Vex pick_up on examination: +40pp

55% → 95%. Same mechanism — pick_up is in Vex's preferred list at +2.

Ember social anyAct: +45pp, look +70pp

35% → 80% anyAct; 0% → 70% look. The "STRICTLY prefers `examine`, `look`, `use`" clause produces a strong social-turn action push.

look aggregates 6× across all cells

Per-cell rate sum: 70% → 485% (+415pp). Pip exploration goes 10% → 75%, Vex exploration 0% → 140% (multi-emit). The model strongly takes "look is preferred" as a directive.

Vex use dips: −30pp on objective

100% → 70%. use sits at +1 bias for Vex — between the preferred (+2) and avoided (−1) thresholds, so it doesn't appear in either list. Vex's preferred picks crowd it out. Lowering the preferred threshold to +1 would re-include it.
